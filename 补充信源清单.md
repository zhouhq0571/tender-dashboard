## 使用规则

### 1. 来源识别与去重机制（★★★ 2026-06-24 强化）

**当用户发来一个 URL 时（所有场景适用）：**
1. **提取域名**：从 URL 中提取域名
2. **与信源清单比对**：
   - 读取 `/Users/zhouhq/Documents/kimi/workspace/bidding-daily/金融机构采购平台清单_v1.md`（主清单）
   - 读取 `/Users/zhouhq/Documents/kimi/workspace/bidding-daily/补充信源清单.md`（补充清单）
3. **判断结果**：
   - 如果域名已存在于任一清单 → 告知用户"此来源已收录，无需重复发送"
   - 如果域名不存在 → **自动添加到本清单的「补充网站信源」表**，告知用户"新来源已记录，以后会纳入搜索范围"
4. **记录要求**：新增来源必须同时记录「来源名称」、「来源类型」、「URL/域名」、「所属机构」、「发现时间」、「纳入原因」

**当用户提到一个微信公众号时**：提取公众号名称，与补充清单比对
- 如果已存在 → 告知用户"此公众号已收录"
- 如果不存在 → 自动添加到「补充微信公众号信源」表

**此规则适用于所有场景（定时任务、手动更新、对话交互），确保用户提供的每一个有价值链接都被纳入常态化信源。**

### 2. 定时任务搜索优先级
```
搜索阶段优先级：
1. 主清单（金融机构采购平台清单_v1.md）→ 各机构自有采购平台、官网栏目
2. 补充清单（本文件）→ 补充网站、公众号、第三方平台
3. 全网增量搜索 → kimi_search_v2 通用搜索
```

### 3. 搜索关键词组合（使用补充信源时）
- **针对补充网站**：`site:域名 招标 2026年` 或 `site:域名 采购 2026年`
- **针对微信公众号**：`公众号名称 招标 2026年` 或 `公众号名称 采购公告`
- **针对第三方平台**：`平台名称 银行 招标 2026年`

### 4. 更新原则
- 新增来源必须同时记录「纳入原因」和「所属机构」
- 不收录非金融机构来源（如普通企业、政府单位、医院等）
- 不收录已失效或长期不更新的来源（超过6个月无招标信息）
- 定期清理（每季度审查一次）
