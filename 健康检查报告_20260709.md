| 2026-07-07 12:10 | `git push origin gh-pages`（从本地 gh-pages 推送） | 远程 = v48 ❌ |
| 2026-07-07 12:15 | 检查 hstender.cn 发现回退到 v48 | 回退确认 |

| 2026-07-07 至今 | 正常更新至 v82 | 稳定运行 ✅ |

### 根因

1. **直接原因**：手动执行 `git push origin gh-pages` 推送了本地停留在 v48 的 gh-pages 分支
2. **深层原因**：
   - deploy.sh 只更新远程 gh-pages，本地 gh-pages 始终停留在 v48（2026-06-28）
   - 2026-06-28 在 gh-pages 分支上做了"补充信源清单更新"操作，导致该分支长期停留旧版本

### 已实施的修复措施

1. ✅ **deploy.sh 分支保护**：必须在 main 分支执行，否则报错退出
2. ✅ **自动删除本地 gh-pages**：部署前自动删除，防止误推
3. ✅ **明确推送路径**：`git push origin main:gh-pages --force`，从 main 推送而非 HEAD
4. ✅ **版本验证**：部署后验证 GitHub Pages 和自定义域名版本一致性
